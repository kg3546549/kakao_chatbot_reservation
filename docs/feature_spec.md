# 3. 기능 명세

## 3.1 핵심 기능 (필수 구현)

| 기능명 | 기능 설명 | 구현 방법 (간략) |
|--------|-----------|-----------------|
| 알림 감지 및 명령어 처리 | 카카오톡 알림을 실시간으로 수신하여 `/예약`, `/취소`, `/조회` 등의 명령어를 파싱하고 자동 답장 전송 | Android `NotificationListenerService` + Flutter MethodChannel (`sendReply`) |
| 예약 CRUD 관리 | 날짜별 항목에 닉네임을 등록·삭제하고 중복 방지, 정원 초과 체크 포함 | SQLite (`sqflite`) — `reservations` 테이블, Provider 상태 관리 |
| 항목 설정 | 예약 항목 이름·최대 정원·응답 템플릿을 추가·수정·삭제; 삭제 시 예약 데이터 연쇄 삭제 | SQLite (`items` 테이블), 외래 키 CASCADE |
| 방 유형 분류 | 카카오톡 채팅방을 예약방·관리자방·일반방으로 분류하여 명령어 처리 범위와 관리자 전용 명령어 권한 제어 | SQLite (`rooms` 테이블), BotProvider 명령어 분기 로직 |
| 대시보드 현황 표시 | 날짜 선택 후 항목별 예약 현황(정원 대비 인원, 명단)을 카드 형태로 실시간 표시; 권한·배터리 최적화 경고 포함 | Flutter UI + Provider (`notifyListeners`), `connectivity_plus` |
| 포그라운드 서비스 유지 | 앱이 백그라운드·도즈 모드에 있어도 알림 감지 서비스가 종료되지 않도록 5분 주기 워치독 동작 | `ForegroundKeepAliveService` (START_STICKY), 배터리 최적화 예외 요청 |
| 설정 | 명령어 단어 커스터마이징, 영업일 기준 초기화 시각 설정, 로그 조회 | `SharedPreferences` (명령어·초기화 시각), SQLite (`logs` 테이블) |

## 3.2 선택 기능 (추가 구현)

| 기능명 | 기능 설명 | 구현 방법 (간략) |
|--------|-----------|-----------------|
| 클라우드 백업 / 복원 | SQLite 데이터를 클라우드에 백업하고 기기 변경 시 복원; 현재는 앱 삭제 시 데이터 유실 | Firebase Firestore 또는 Google Drive API |
| 대기열(웨이팅) 관리 | 정원 초과 시 대기 등록 후 취소 발생 시 자동으로 대기자에게 예약 전환 및 알림 | SQLite `waitlist` 테이블 추가, BotProvider 대기 순번 로직 |
| 데이터 내보내기 | 예약 이력을 CSV 또는 Excel 파일로 내보내 외부 관리 가능 | `csv` 패키지 + `share_plus`로 파일 공유 |
| 다크 모드 | 앱 전체 UI를 다크 테마로 전환; 시스템 설정 연동 또는 수동 토글 | Flutter `ThemeMode` + `SharedPreferences`로 설정 저장 |
| 통계 고도화 | 월별 예약 추이, 항목별 인기도 랭킹, 요일별 패턴 분석 등 심화 차트 제공 | `fl_chart` 확장, SQLite 집계 쿼리 (`GROUP BY`, `strftime`) |
| 예약 승인 시스템 | 관리자가 예약 요청을 수동으로 승인·거절할 수 있는 워크플로 추가 | SQLite `status` 컬럼 추가 (`pending` / `approved` / `rejected`), 관리자방 전용 명령어 |
| 정기 예약 / 반복 예약 | 매주 특정 요일에 자동으로 예약을 생성하는 반복 예약 기능 | `flutter_local_notifications` + 주기적 스케줄러 |

---

### 보충 설명

- **알림 감지**가 이 앱의 핵심 엔진으로, 카카오톡 `RemoteInput` 액션을 캐싱해두었다가 답장 전송에 재사용합니다.
- **방 유형 분류**는 실질적인 권한 제어 역할을 하여 일반방에서는 명령어를 완전히 무시하고, 관리자방에서만 `/초기화`, `/세팅최대` 등의 명령어가 실행됩니다.
- **포그라운드 서비스**는 Android의 도즈 모드 및 앱 킬에 대응하기 위한 필수 구현 항목입니다.
